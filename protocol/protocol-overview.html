设计原则

Agent Control Center 是系统型应用，原生能力（后台、通知、网络、权限、硬件）是核心竞争力。双原生方案让两端各用平台最佳实践，只在协议层统一。

📋

JSON Schema 是唯一事实来源

数据模型的字段名、类型、枚举值以 schema 文件为准，两端手写映射，不搞代码生成。

🔗

传输格式跨平台一致

HTTP/SSE/WebSocket/MCP 的线上字节流格式必须完全相同，两端可互操作。

🔐

加密格式跨平台兼容

AKS: 前缀（静态存储）和 AH1: 前缀（传输加密）在两端行为一致。

📐

向后兼容的变更策略

新增字段加 default，移除字段先 deprecated，枚举只能新增值，传输变更两端同步。

跨平台一致性约束

MessageRole 大小写差异：HTTP API 中使用小写（user / assistant / system / tool），WebSocket 帧中使用枚举名首字母大写（User）。两端序列化时必须做映射，不能混用。
加密前缀格式：AKS: 用于 API Key 静态存储（Keystore/Keychain），AH1: 用于消息 E2E 传输加密。两个前缀格式不可混用，iOS 实现必须支持相同的解密逻辑。
sessionId 贯穿全链路：WebSocket 传输中 sessionId 由服务端维护会话状态，客户端不重放历史。HTTP 传输中 sessionId 仅用于客户端侧滑动窗口历史管理（上限 20 条），服务端无状态。
systemPrompt 当前未注入：Android HTTP 传输的请求体只发送 model / messages / stream / temperature / max_tokens，未注入 systemPrompt。协议层保留此字段，iOS 实现可选择注入或保持一致。
