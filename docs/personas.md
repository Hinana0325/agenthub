# Agent Control Center 用户画像

> 配套文档：[`product-strategy.md`](product-strategy.md) · 最后更新：v4.8.0（2026-07-23）
> 用途：驱动功能优先级决策，而非技术债驱动。每个新特性立项前先回答"服务于哪个 persona 的哪个痛点"。

## 一、为什么需要 persona

`product-strategy.md` 定义了三层目标用户（核心层 / 扩展层 / 泛化层）。本文档将这三层具象化为 3 个可操作的 persona，让 Sprint 规划、功能取舍和文案基调有共同锚点。

---

## 二、核心 Persona

### Persona 1：自部署开发者 Alex

| 维度 | 描述 |
|:-----|:-----|
| **画像** | 28–40 岁，后端 / 全栈开发者，家有 NAS 或高配桌面跑 Ollama / LM Studio，ComfyUI 出图，同时订阅 OpenAI / Claude API |
| **技术水平** | 高。能改 YAML、能读源码、能配反向代理，命令行原生用户 |
| **设备** | Android 旗舰（Pixel / 小米旗舰）+ iPad Pro，重视双端一致 |
| **核心目标** | 1) 在手机上统一操控本地 + 云端异构 Agent；2) 保证 API Key 和本地端点不出设备；3) 把 ComfyUI 出图 / Ollama 推理接入日常对话流 |
| **痛点** | ① 本地 Agent（Ollama/ComfyUI）没有像样的移动客户端，只能 SSH 或开网页；② 多个 Key 散落在备忘录，怕泄露；③ 云端和本地 Agent 无法在同一次对话里协作；④ 担心厂商客户端把对话上传云端做训练 |
| **当前替代方案** | OpenAI 官方 App + Ollama WebUI + 备忘录存 Key + 服务器 SSH，切来切去 |
| **ACC 如何帮** | 8 种 AgentType 单 App 接入（含 Ollama / LM Studio / ComfyUI 本地类型）；`AKS:` 静态加密 + `AH1:` 端到端传输，Key 不出设备；双原生 Android + iOS 体验一致；后续端侧推理（MLX/MediaPipe）让"本地"从"转发到服务"升级为"App 内推理"，满足其对隐私和控制的极致要求 |
| **驱动特性** | 端侧推理（v5.0.0 P1）、E2E 加密、本地 Agent 接入、Agent 配置自由度 |

---

### Persona 2：多 Agent 重度用户 Sam

| 维度 | 描述 |
|:-----|:-----|
| **画像** | 30–45 岁，产品经理 / 研究员 / 内容创作者，付费订阅 Claude + GPT + Gemini，每天在 3+ 个 AI 工具间切换 |
| **技术水平** | 中高。会用 API、能写简单脚本，但不维护服务器，不碰本地模型部署 |
| **设备** | iPhone 为主力 + Mac，偶尔 Android |
| **核心目标** | 1) 用一个入口调度多个云端 Agent；2) 把"让 GPT 翻译 → Claude 润色 → Gemini 校验事实"这类跨 Agent 流程自动化；3) 移动端也能跑这些流程，不只绑在电脑前 |
| **痛点** | ① 跨 Agent 协作全靠手动复制粘贴；② n8n / Zapier 面向 SaaS 集成，不面向 AI Agent 编排，搭不起来；③ 现有工作流跑完即丢，没法复用和迭代；④ 没法可视化看到编排结构，改流程靠脑补 |
| **当前替代方案** | 浏览器多标签页 + 手动复制粘贴，或用 ChatGPT 自定义 GPT 凑合，跨 Agent 流程基本靠人肉 |
| **ACC 如何帮** | DAG 工作流引擎支持跨 Agent 编排（云端 Agent 间串联）；v4.9.0 Workflow 持久化 + 执行历史解决"跑完即丢"；v5.0.0 可视化拖拽编辑器让编排"所见即所得"；双端原生让流程在手机上也能跑 |
| **驱动特性** | Workflow 持久化（v4.9.0 P0）、可视化编辑器（v5.0.0 P0）、Marketplace 工作流模板、并行分支 + 条件节点（v5.1.0） |

---

### Persona 3：效率工具探索者 Riley（扩展层，次要）

| 维度 | 描述 |
|:-----|:-----|
| **画像** | 25–35 岁，运营 / 设计 / 自由职业者，被各种"AI 周报生成器""AI 海报工具"吸引，但对 Agent 类型不感兴趣 |
| **技术水平** | 低。不会配 API，不关心模型名，只想点一下出结果 |
| **设备** | iPhone 为主，对"开箱即用"敏感 |
| **核心目标** | 1) 不懂技术也能用上多步骤 AI 流程；2) 用场景化模板（"写周报""做海报""翻译校对"）一键启动；3) 结果能直接分享或导出 |
| **痛点** | ① 现有 AI App 功能单一，无法组合多步骤任务；② 配置门槛高，看到"API Key""端点"就劝退；③ 模板工具固化，稍微改需求就不灵 |
| **当前替代方案** | 单点 AI 工具 + 手动拼流程，或找技术朋友帮忙配 |
| **ACC 如何帮** | Marketplace 提供社区 Agent 与工作流模板（一键订阅）；v5.1.0+ 模板市场降低使用门槛；当前阶段 ACC 对 Riley 偏重，需等模板生态成熟后才能真正服务 |
| **驱动特性** | Marketplace 可订阅（v4.9.0 P1）、工作流模板、场景化入口、降低配置心智（默认值 + 引导） |

## 三、Persona 与版本路线图映射

| 路线图项 | 主服务 Persona | 优先级理由 |
|:---------|:--------------|:-----------|
| Workflow 持久化 + 历史（v4.9.0 P0） | Sam | 解决"跑完即丢"核心痛点 |
| 填平 iOS CollaborationManager（v4.9.0 P0） | Sam / Alex | 双端一致是产品承诺 |
| Marketplace 可订阅（v4.9.0 P1） | Sam / Riley | 提升留存与模板复用 |
| 可视化拖拽编辑器（v5.0.0 P0） | Sam | 用户感知最强的差异化，分水岭特性 |
| 端侧推理 MVP（v5.0.0 P1） | Alex | 隐私 + 离线杀手锏，Alex 的核心诉求 |
| RAG Agent（v5.0.0 P2） | Sam | 从"聊天"走向"知识工作" |
| MCP 工具市场（v5.1.0 P1） | Alex / Sam | Agent 能力扩展机制 |
| 轻量团队协作（v5.1.0 P2） | Sam | 从个人走向团队 |

**取舍原则**：当资源冲突时，**优先服务 Alex + Sam（核心层）**。Riley 的需求由模板生态成熟后自然承接，不为本期版本牺牲核心层体验。
